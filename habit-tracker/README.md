# 콤보 챌린지

> 콤보를 깨지 않는 게 가장 강력한 동기다.

기간을 정해놓고, 그 기간 동안 좋은 습관(또는 안 하기로 한 행동)에 콤보를 쌓는 개인용 해빗 트래커. 한 번 실패하면 콤보가 0으로 리셋되니, 콤보를 깨지 않으려는 마음으로 행동을 유지하게 됩니다.

**🌐 배포 주소**: https://grkang333-glitch.github.io/994257/habit-tracker/

---

## 사용 흐름

1. **챌린지 설정** — 이름(예: "30일 건강 챌린지"), 시작일, 목표 기간(일) 입력
2. **태스크 등록** — "삼시세끼 챙겨먹기", "야식 안 먹기" 같은 태스크 추가. 각 태스크는 **체크포인트(슬롯)** 들로 구성 (예: 삼시세끼는 아침/점심/저녁 슬롯)
3. **매일 체크인** — 오늘의 슬롯마다 **달성** / **실패** 버튼 클릭
4. **콤보 누적** — 슬롯을 시간순으로 성공시킬 때마다 콤보 +1. 실패 누르면 0으로 리셋
5. **격려 메시지** — 매 체크마다 코치 톤의 메시지가 표시됨

---

## 콤보 규칙

각 태스크의 **모든 슬롯**을 챌린지 시작일부터 오늘까지 시간순으로 일렬로 펼친 다음, **마지막 "실패" 이후의 연속 "달성" 개수**가 현재 콤보입니다.

### 예시

**태스크 A — 삼시세끼 (슬롯: 아침/점심/저녁)**

| 날짜 | 아침 | 점심 | 저녁 | 누적 콤보 |
|---|---|---|---|---|
| 5/26 | ✅ | ✅ | ✅ | 3 |
| 5/27 | ✅ | ❌ | ✅ | (3→4→0→1) |
| 5/28 | ✅ | ✅ | ✅ | (1→2→3→4) |

5/27 점심에 실패해서 콤보 리셋, 이후 다시 5번 연속 성공해서 현재 **5콤보**.

**태스크 B — 운동하기 (슬롯: 완료)**

슬롯이 하나뿐이면 **연속 일수가 곧 콤보**가 됩니다. 3일 연속 운동 = 3콤보.

### 미입력의 의미

아직 체크 안 한 슬롯(예: 오늘 저녁 슬롯을 아직 안 누름)은 **콤보를 유지**합니다. 미래 시점이라 판단해서 그렇습니다. 즉 콤보가 깨지는 건 명시적으로 "실패"를 눌렀을 때만입니다.

### 토글 해제

같은 버튼을 한 번 더 누르면 그 슬롯이 미입력 상태로 돌아갑니다. 잘못 눌렀을 때 쓰세요.

## 화면 구성

### 상단 Hero
- 챌린지 이름, 시작일 → 종료일, 총 일수
- **남은 일수**(큰 숫자), 진행률 바

### 오늘의 체크인
- 등록된 모든 태스크와 오늘의 슬롯 버튼
- 각 태스크 우측에 "현재 N콤보" 표시

### 콤보 현황
- 태스크별 **현재 콤보**, **최고 콤보**, **남은 기간 동안 도달 가능한 최대 콤보**
- "남은 기간 최대 N콤보 가능" 메시지는 콤보가 깨졌을 때도 다시 도전 의욕을 살리는 역할

### 코치 메시지
- 달성 시: `"좋습니다. [태스크·슬롯] 달성! N콤보 진입했습니다. 다음 [슬롯]까지 이어가면 M콤보입니다."`
- 실패 시: `"콤보가 리셋됐습니다. 자책은 짧게. 남은 N일 동안 최대 M콤보까지 다시 만들 수 있습니다."`

## Google Sheets 동기화

본인 전용 백엔드 URL과 닉네임이 `app.js` 상단 상수에 하드코딩되어 있어서, 어느 기기에서 페이지를 열어도 자동으로 같은 시트와 동기화됩니다. 설정 화면에는 동기화 관련 입력 없음.

### 동기화 동작

- **저장**: 슬롯 체크 등 변경 발생 → 1.5초 디바운스 후 시트로 POST
- **불러오기 (자동)**: 페이지 로드 시 / 다른 탭 갔다 돌아왔을 때 / 60초마다(창 보일 때)
- **충돌 정책**: last-write-wins (두 기기에서 동시에 다른 슬롯 누르면 늦게 저장된 쪽이 이김)
- **안전 장치**: 로컬에서 방금(3초 이내) 변경했거나 저장 대기 중이면 자동 불러오기 건너뜀

상단에 동기화 상태 점이 표시됨: `저장 대기` → `동기화 중` → `동기화됨`(초록) / `동기화 실패`(빨강)

### 이벤트 로그

슬롯 변경마다 시트의 `events` 탭에 한 줄씩 누적됩니다. 나중에 통계를 보거나 패턴을 분석할 때 사용 가능:

```
timestamp | userId | date | taskId | taskName | slot | status
```

## 데이터 모델

### localStorage (브라우저)

`habit-tracker-v1` 키에 JSON으로 저장:

```js
{
  challenge: { title, startDate: 'YYYY-MM-DD', days },
  tasks: [
    { id, name, slots: ['아침','점심','저녁'] },
    ...
  ],
  logs: {
    'YYYY-MM-DD': {
      [taskId]: {
        [slotName]: 'ok' | 'fail'
      }
    }
  },
  lastTouched: { [taskId]: epochMs }
}
```

### Google Sheet (선택)

- **`states` 탭**: `userId | state(JSON 한 줄) | updated` — 사용자별 1행
- **`events` 탭**: `timestamp | userId | date | taskId | taskName | slot | status` — append-only

### 다중 사용자

`userId` 필드로 네임스페이스를 분리합니다. 같은 시트를 여러 명이 써도 데이터는 섞이지 않습니다. (단 시트 소유자는 모든 사용자의 데이터를 볼 수 있음)
